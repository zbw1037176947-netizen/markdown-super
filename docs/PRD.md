> 版本：v1.0.0

> 状态：待确认

---

## 一、产品定位

VS Code 上的**一站式 Markdown 阅读与编辑体验**。目标是替代用户同时安装 3-4 个 Markdown 插件的现状，提供一个丝滑、美观、功能完整的单一插件。

**核心差异化**：预览主题独立可控 + 滚动同步 + 图表原生支持 + 开箱即用。

**目标用户**：日常用 Markdown 写技术文档、笔记、博客的开发者。

**发布计划**：自用打磨 → 上架 VS Code Marketplace。

## 二、功能清单

### P0 — 核心体验（本期必做）

#### 2.1 分屏预览 + 双向滚动同步

- 左侧源码编辑器，右侧预览面板
- **编辑器 → 预览**：光标所在行对应的内容滚动到预览视口中央
- **预览 → 编辑器**：点击预览中的段落/标题，源码编辑器跳转到对应行
- 打字时预览实时更新（50ms 防抖，万行以内不卡顿）

#### 2.2 独立主题系统

两种模式，通过预览面板右上角图标一键切换：

| 模式 | 行为 |
|------|------|
| **跟随 VS Code** | 预览背景/文字色跟随 VS Code 当前主题（暗色/亮色） |
| **阅读模式（亮色）** | 固定白底（#fff）+ 深灰文字（#1a1a1a）+ 无衬线字体 + 加大行距，不受 VS Code 主题影响 |

#### 2.3 图片支持

| 功能 | 交互 |
|------|------|
| 显示图片 | 本地相对路径、绝对路径、网络 URL 均正确渲染 |
| 粘贴截图 | 在源码编辑器中 Ctrl+V 粘贴剪贴板图片 → 自动保存到 `./assets/` 目录 → 插入 `![](./assets/xxx.png)` |
| 拖拽插入 | 拖拽图片文件到编辑器 → 同上逻辑 |

图片保存目录可配置（默认 `./assets/`），文件名格式 `YYYYMMDD-HHmmss-随机4位.png`。

#### 2.4 图表渲染

| 类型 | 语法 | 渲染方式 |
|------|------|----------|
| **Mermaid** | ` ```mermaid ` | 浏览器端渲染（已实现） |
| **PlantUML** | ` ```plantuml ` | 本地 WASM 渲染（离线可用，无隐私泄露） |
| **Markmap** | ` ```markmap ` 或自动从标题生成 | markmap-lib 浏览器端渲染 |

#### 2.5 大纲侧边栏（TOC）

- 在 VS Code **侧边栏**注册一个 Tree View，显示当前 Markdown 文件的标题树
- 层级缩进（H1 → H2 → H3...）
- 点击标题 → 源码编辑器跳转到对应行 + 预览滚动到对应位置
- 文档内容变化时自动更新

#### 2.6 代码块增强

- 语法高亮（highlight.js，已实现）
- 右上角**复制按钮**，点击后显示"Copied!"反馈
- 显示语言标签（如 `python`、`javascript`）
- 行号显示（可配置开关）

### P1 — 体验打磨（本期尽量做）

#### 2.7 Front Matter 渲染

YAML front matter（`---` 包裹）渲染为信息卡片，而非原始文本：

```
┌─ Front Matter ─────────────────┐
│ title:    My Article           │
│ date:     2026-04-10           │
│ tags:     markdown, vscode     │
│ author:   zhangsan             │
└────────────────────────────────┘
```

#### 2.8 字数统计 & 阅读时长

在预览面板底部状态栏或 VS Code 状态栏显示：

```
📄 1,234 字  |  🕐 约 5 分钟阅读
```

中文按字计数，英文按词计数。阅读速度：中文 300 字/分钟，英文 200 词/分钟。

#### 2.9 搜索高亮

在预览面板内提供 Ctrl+F 搜索，匹配文本高亮显示（VS Code 自带搜索只作用于源码编辑器）。

#### 2.10 快捷格式化

在源码编辑器中提供快捷键：

| 快捷键 | 功能 |
|--------|------|
| Ctrl+B | 加粗 `**text**` |
| Ctrl+I | 斜体 `*text*` |
| Ctrl+K | 插入链接 `[text](url)` |
| Ctrl+Shift+K | 插入图片 `![alt](url)` |
| Ctrl+Shift+C | 行内代码 `` `code` `` |

### P2 — 前瞻功能（预留架构，后续迭代）

- **WYSIWYG 编辑模式**：基于 Milkdown 的所见即所得，可切换
- **实时协作**：基于 Y.js/CRDT
- **自定义 CSS 主题**：用户可导入自己的预览样式
- **插件系统**：允许第三方扩展渲染器

## 三、非功能需求

### 3.1 性能指标

| 指标 | 目标 |
|------|------|
| 首次预览渲染 | < 500ms（3000 行文件） |
| 增量更新延迟 | < 100ms（打字到预览刷新） |
| 滚动帧率 | 60fps 无掉帧 |
| 内存占用 | < 150MB（含 Mermaid） |
| Mermaid 渲染 | < 2s/图（复杂图表） |

### 3.2 兼容性

- VS Code >= 1.90.0（2024 年中）
- 支持 Windows / macOS / Linux
- 支持 Remote SSH / WSL / Codespaces

### 3.3 Marketplace 准备

- 清晰的 README + 功能截图 + GIF 演示
- CHANGELOG.md 规范维护
- MIT 开源协议
- CI/CD 自动发布（GitHub Actions）

---

## 四、技术约束

- 构建工具：esbuild（双目标 Node + Browser）
- 预览渲染：markdown-it + 后处理器（Mermaid/PlantUML/Markmap）
- 图表大库（Mermaid ~8MB）必须延迟加载，不影响首屏
- Webview 与扩展通过 postMessage 通信
- CSP 策略严格，不允许 eval / inline script
